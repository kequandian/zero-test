# render-mdpdf-service

A standalone Rust-based HTTP service that provides Markdown to PDF conversion capabilities via HTTP/SSE API. It integrates `render-pdf-mdpdf` (Node.js + mdpdf/puppeteer) internally.

## Features

- **Rust/Axum HTTP Server** - High-performance server on port 39090
- **Synchronous Rendering** - Direct PDF conversion with immediate response
- **Asynchronous Rendering** - Job-based conversion with SSE progress streaming
- **Docker Support** - Multi-stage builds for easy deployment
- **k3s Integration** - Build via k3s-proxy for cluster deployment

## Quick Start

```bash
cd zero-test/render-mdpdf-service
cargo run
```

Default: `http://127.0.0.1:39090`

## Docker Build & Run

### Local Build

```bash
cd zero-test/render-mdpdf-service

# Build image
docker build -t render-mdpdf-service:latest .

# Run container
docker run --rm -p 39090:39090 \
  -v "$(pwd)/output:/app/output" \
  render-mdpdf-service:latest
```

Container `WORKSPACE_ROOT=/app`, output writes to `/app/output`.

### Build via k3s-proxy

```bash
cd zero-test/render-mdpdf-service/build/cli
python3 build-via-k3s-proxy.py
```

After successful build, image is available at `registry.k3s.local/render-mdpdf-service:latest`.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Health check |
| POST | `/api/mdpdf/render` | Synchronous PDF conversion |
| POST | `/api/mdpdf/render/async` | Asynchronous PDF conversion |
| GET | `/api/mdpdf/jobs/{id}` | Query job status |
| GET | `/api/mdpdf/jobs/{id}/events` | SSE job events stream |
| GET | `/api/mdpdf/files/{filename}` | Download converted PDF |

## Request Examples

### Synchronous (markdown string)

```bash
curl -X POST "http://127.0.0.1:39090/api/mdpdf/render" \
  -H "Content-Type: application/json" \
  -d '{
    "markdown":"# Hello\n\nGenerated by remote service"
  }'
```

**Response:**
```json
{
  "output_path": "/app/output/abc-123-def.pdf",
  "download_url": "/api/mdpdf/files/abc-123-def.pdf",
  "duration_ms": 1234
}
```

**Download PDF:**
```bash
curl -O "http://127.0.0.1:39090/api/mdpdf/files/abc-123-def.pdf"
```

### Asynchronous (markdown string)

```bash
curl -X POST "http://127.0.0.1:39090/api/mdpdf/render/async" \
  -H "Content-Type: application/json" \
  -d '{
    "markdown":"# Async render\n\nwith SSE"
  }'
```

**Response:**
```json
{
  "job_id": "abc-123-def",
  "status_url": "/api/mdpdf/jobs/abc-123-def",
  "events_url": "/api/mdpdf/jobs/abc-123-def/events",
  "download_url": "/api/mdpdf/files/abc-123-def.pdf"
}
```

### SSE Subscribe

```bash
curl -N "http://127.0.0.1:39090/api/mdpdf/jobs/<job_id>/events"
```

## Request Body Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `markdown` | string | No* | Markdown text (mutually exclusive with `markdown_path`) |
| `markdown_path` | string | No* | Path to markdown file (relative to workspace root) |
| `css_path` | string | No | Custom CSS file path |
| `header` | string | No | Page header |
| `footer` | string | No | Page footer |

*Either `markdown` or `markdown_path` must be provided.

**Note**: Output PDF is automatically saved as `output/<job_id>.pdf`. Use the `download_url` from the response to download the file.

## Response Fields

### Sync Response

| Field | Type | Description |
|-------|------|-------------|
| `output_path` | string | Full file path on server |
| `download_url` | string | HTTP download URL |
| `duration_ms` | number | Conversion time in milliseconds |

### Async Response

| Field | Type | Description |
|-------|------|-------------|
| `job_id` | string | Unique job identifier (UUID) |
| `status_url` | string | URL to query job status |
| `events_url` | string | SSE endpoint for live updates |
| `download_url` | string | HTTP download URL (available after completion) |

### Job Status Response

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Job ID |
| `status` | string | `queued`, `running`, `succeeded`, or `failed` |
| `output_path` | string/null | Output file path (null if not completed) |
| `download_url` | string/null | Download URL (null if not completed) |
| `error` | string/null | Error message (null if no error) |
| `created_at_ms` | number | Job creation timestamp |
| `updated_at_ms` | number | Last update timestamp |
| `duration_ms` | number/null | Job duration in milliseconds |

## Downloading PDF Files

After conversion completes, use the `download_url` from the response:

```bash
# Sync response example
curl "http://127.0.0.1:39090/api/mdpdf/files/abc-123-def.pdf" -o output.pdf

# Query job status for async, then download
curl "http://127.0.0.1:39090/api/mdpdf/jobs/<job_id>" | jq -r '.download_url'
curl "http://127.0.0.1:39090$(curl -s "http://127.0.0.1:39090/api/mdpdf/jobs/<job_id>" | jq -r '.download_url')" -o output.pdf
```

### Security Notes

- Only files in the `output/` directory are accessible
- Filename must not contain `..`, `/`, or `\` (path traversal protection)
- Files are served with `Content-Disposition: attachment` header
